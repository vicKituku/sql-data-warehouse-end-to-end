# Data Warehousing and Analytics Project

## Overview

This project builds a modern data warehouse using **PostgreSQL** to consolidate ERP and CRM data for analytics and reporting.

The goal is to clean, integrate, and model business data into a structure that supports SQL-based analysis and informed decision-making.

## Specifications

### Data Sources

- **ERP data**: sales, orders, products, and operational records
- **CRM data**: customer and relationship management records

Source files are provided as CSV files.

### Data Quality

Data is cleaned before analysis by:

- Removing duplicates
- Handling missing values
- Standardizing formats
- Fixing inconsistent records
- Validating key fields

### Data Integration

ERP and CRM data are combined into a single analytics-ready model designed for business reporting.

The model supports analysis across:

- Customers
- Products
- Sales
- Orders
- Time periods

### Scope

This project focuses on the **latest dataset only**.

No historical tracking or historization is included.

### Documentation

The project includes documentation for:

- Data sources
- Data cleaning rules
- Data model structure
- Table relationships
- Analytical queries

## BI: Analytics and Reporting

SQL-based analytics are developed to generate insights into:

### Customer Behavior

- Top customers by revenue
- Customer purchase patterns
- Customer segmentation

### Product Performance

- Best-selling products
- Revenue by product/category
- Low-performing products

### Sales Trends

- Sales over time
- Revenue trends
- Order volume patterns

## Outcome

This project delivers:

- A centralized PostgreSQL data warehouse
- Clean and integrated ERP/CRM data
- A simple analytical data model
- SQL reports for business insights
- Documentation for business and analytics users
